# Wireframes: Prompt Template Architecture v1

**Sprint:** prompt-template-architecture-v1
**Version:** 3.0 (Architecturally Complete)
**Created:** 2026-01-21
**Status:** Ready for User Approval

---

## Architectural Decisions (UX Chief Approved)

### Naming
- **Schema:** `OutputTemplate` / `OutputTemplatePayload`
- **UI Section:** "Output Templates" in Experience Console
- **Rationale:** Broad enough for documents, code, configs; specific enough to understand

### State Management
```
status: 'active' | 'archived' | 'draft'
```
| State | In Selection? | Editable? | Badge |
|-------|---------------|-----------|-------|
| `draft` | No | Yes | Yellow dot |
| `active` | Yes | Yes | Green dot |
| `archived` | No | View only | Gray dot |

### Provenance
```typescript
source: 'system-seed' | 'user-created' | 'imported' | 'forked'
forkedFromId?: string
previousVersionId?: string
```

### Future Pattern (Documented, Not In Scope)
Singleton-Instance pattern where `OutputStyle` singleton contains references to `OutputTemplate` instances. See Notion: [Architectural Note](https://www.notion.so/2ef780a78eef8159b788c08119cc9632)

---

## Design Principles Applied

| Pattern | Description | Reference |
|---------|-------------|-----------|
| **Objects and Inspectors** | Side-by-side list + inspector | `InspectorPanel.tsx`, `ModelCard.tsx` |
| **Configuration vs Usage** | Config in Experience Console, usage in /explore | Architecture principle |
| **Console Factory** | All consoles use `createBedrockConsole` | `ExperienceConsole/` |
| **Fork for System Seeds** | Users fork, not edit, system templates | Provenance principle |
