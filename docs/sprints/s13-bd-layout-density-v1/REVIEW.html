Sprint Summary

Implemented a declarative layout density system for json-render that eliminates CSS hacks and enables consistent spacing configuration across all dashboards.

Core Deliverables

  • ✓
    LayoutConfig TypeScript Interface

    Three density presets: compact, comfortable, spacious with optional token overrides

  • ✓
    DENSITY_PRESETS Object

    Tailwind class mappings for each density level based on 4px grid

  • ✓
    Renderer Integration

    Added layout prop with LayoutContext for child component access

  • ✓
    CSS Hack Migration

    Eliminated all [&_.json-render-root]:space-y-* patterns

Density Presets

Density Container Padding Section Gap Card Gap Use Case
compact p-3 space-y-2 gap-2 Dense data displays, dashboards
comfortable p-6 space-y-4 gap-4 Default, balanced spacing
spacious p-8 space-y-6 gap-6 Readability-focused layouts

Usage Guide

Basic Usage

import { Renderer } from '@surface/components/modals/SproutFinishingRoom/json-render';

// Default comfortable density
<Renderer tree={tree} registry={registry} />

// Explicit density selection
<Renderer tree={tree} registry={registry} layout={{ density: 'compact' }} />

// With token override
<Renderer
  tree={tree}
  registry={registry}
  layout={{ density: 'spacious', containerPadding: 'p-0' }}
/>

Accessing Layout in Child Components

import { useLayoutContext } from '@surface/components/modals/SproutFinishingRoom/json-render';

function MyComponent() {
  const layout = useLayoutContext();
  return <div className={layout?.cardGap}>...</div>;
}
