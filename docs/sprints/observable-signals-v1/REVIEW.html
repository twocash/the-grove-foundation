Key Features Delivered

  • 8 Event Types: sprout_viewed, sprout_retrieved, sprout_referenced, sprout_searched, sprout_rated, sprout_exported, sprout_promoted, sprout_refined
  • Full Provenance Tracking: Every event captures lens, journey, hub, and query context
  • Signal Aggregation: Pre-computed metrics for view, utility, diversity, and quality scores
  • Phase 3 Ready: Advancement eligibility calculated for future auto-tier-advancement
  • Operator Analytics: Usage Signals panel in both Nursery Console and Finishing Room
  • Offline Support: Signal emission queue with localStorage persistence and auto-sync
  • json-render Pattern: Declarative component vocabulary for flexible analytics display

DEX Compliance

Declarative Sovereignty

Event types defined in schema, signal display via json-render catalog. Domain experts can modify analytics view without code changes.

Capability Agnosticism

Pure data capture and aggregation. No model-specific code paths. Works with any AI that produces sprouts.

Provenance as Infrastructure

Full event lineage tracked in EventProvenance. Every signal traces back to lens, journey, hub, and query.

Organic Scalability

Discriminated union pattern for events. JSONB metadata for extensible payloads. New signals additive via schema.

Key Insight

"We're not building analytics. We're building the nervous system for emergent quality."

This sprint establishes the foundation for Phase 3 (Auto-Advancement), Phase 6 (AI Curation), and Phase 7 (Attribution Economy) by creating comprehensive usage signal capture with full provenance tracking.
