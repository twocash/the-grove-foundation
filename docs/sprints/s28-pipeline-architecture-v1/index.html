Sprint Overview

The Grove pipeline has three configuration layers (research-agent-config, writer-agent-config, output-templates) that were designed to compose declaratively, but currently they're disconnected. The UI allows editing these configs, but the pipeline never reads them ‚Äî everything uses hardcoded defaults.

Critical Gaps Identified:

  • config-loader.ts has TODO stubs that return hardcoded defaults instead of querying Supabase
  • research-agent-config is stored but never read during evidence gathering
  • writer-agent-config is stored but never read during document writing
  • OutputTemplate.config field exists but is completely ignored
  • server.js has fallback defaults that override all configuration
  • Provenance tracking doesn't record which config versions produced a document
